### **方法2: 命令行快速连接**
```bash
# 基本连接
remmina -c rdp://192.168.1.7

# 带用户名连接
remmina -c "rdp://username@192.168.1.7"

# 带用户名和域连接
remmina -c "rdp://DOMAIN\\username@192.168.1.7"
```

### **方法3: 使用配置文件**
```bash
# 创建配置文件
cat > ~/.local/share/remmina/192.168.1.7.remmina << EOF

name=192.168.1.7
server=192.168.1.7
protocol=RDP
group=
exec=
execpath=
quality=0
disableautoreconnect=0
username=
password=
domain=
security=
gateway_hostname=
gateway_username=
gateway_password=
gateway_domain=
clientname=
EOF

# 使用配置文件连接
remmina -c ~/.local/share/remmina/192.168.1.7.remmina
```

## 🛠️ 故障排除

### **常见问题及解决**

#### **1. 连接被拒绝**
```bash
# 检查端口
nc -zv 192.168.1.7 3389

# 检查防火墙
sudo ufw status
```

#### **2. 认证失败**
- 确认用户名密码正确
- 检查Windows用户是否有远程桌面权限
- 尝试使用 `计算机名\用户名` 格式

#### **3. 黑屏或连接断开**
- 调整显示质量设置
- 检查网络稳定性
- 尝试禁用声音重定向

#### **4. 证书警告**
- 第一次连接时接受证书
- 或添加证书到信任列表

### **Windows端检查**
1. **开启远程桌面**:
   - 设置 → 系统 → 远程桌面 → 启用
   
2. **防火墙设置**:
   - 控制面板 → Windows Defender 防火墙 → 允许应用
   - 确保"远程桌面"被允许

3. **用户权限**:
   - 设置 → 系统 → 远程桌面 → 选择用户
   - 添加允许远程连接的用户

## 🚀 快速连接脚本

### **一键连接脚本**
```bash
#!/bin/bash
# 保存为 connect_192.168.1.7.sh

TARGET="192.168.1.7"
USERNAME="${1:-}"  # 第一个参数为用户名
PASSWORD="${2:-}"  # 第二个参数为密码

echo "连接到 $TARGET..."

if [ -n "$USERNAME" ] && [ -n "$PASSWORD" ]; then
    # 带用户名密码连接
    remmina -c "rdp://$USERNAME:$PASSWORD@$TARGET"
elif [ -n "$USERNAME" ]; then
    # 只带用户名连接
    remmina -c "rdp://$USERNAME@$TARGET"
else
    # 基本连接
    remmina -c "rdp://$TARGET"
fi
```

### **使用示例**
```bash
# 基本连接
./connect_192.168.1.7.sh

# 带用户名连接
./connect_192.168.1.7.sh "Administrator"

# 带用户名密码连接
./connect_192.168.1.7.sh "Administrator" "password123"
```
